# アプリ概要

**主要機能**
- 🀄 対局結果の記録・管理
- 📊 点数計算（ウマ・オカ対応、同点処理（上家取り・折半））
- 📱 4人以上の成績のサマリ機能

**技術スタック**
- **フレームワーク**: Flutter 3.8+
- **状態管理**: Riverpod
- **データベース**: SQLite
- **ルーティング**: GoRouter

---

# アプリ開発を行うきっかけ

## 課題認識 🤔
- 既存の麻雀管理アプリに不満
  - 複雑すぎる操作性
  - 点数計算の不正確さ
  - データの可視化不足

## 目標 🎯
- **シンプルで使いやすい**麻雀点数管理
- **正確な計算**（上家取り・折半対応）
- **長期データ分析**が可能な設計

---

# 施策

## 開発方針
1. **ユーザビリティ重視**
   - 最小限の入力で最大限の価値提供
   - 直感的なUI/UX設計

2. **技術的堅牢性**
   - モダンな技術スタック採用
   - テスタブルな設計
   - 型安全性の確保

3. **段階的開発**
   - MVPからの段階的機能追加
   - ユーザーフィードバックに基づく改善

# 進捗

## 開発状況 📈
- **総ファイル数**: 30 Dartファイル
- **実装済み機能**:
  - ✅ 基本的な対局記録機能
  - ✅ 点数計算サービス（ウマ・オカ対応）
  - ✅ ゲーム設定画面
  - ✅ データ永続化（SQLite）
  - ✅ 状態管理（Riverpod）

## アーキテクチャ構成
```
lib/
├── models/          # データモデル
├── screens/         # UI画面

└── utils/          # ユーティリティ
```

# 進捗（詳細）

## 実装済みモデル
- `Round`: 対局情報
- `UmaOka`: 点数計算設定
- `SamePointMode`: 同点処理モード
- `Wind`: 風の管理
- `RankData`: 順位データ

## 実装済みサービス
- `PointCalcService`: 点数計算エンジン
- `DatabaseHelper`: データ永続化
- マイグレーション管理

## 画面構成
- ホーム画面
- ゲーム設定画面
- 点数入力画面
- 対局履歴画面

# デモ

## 主要画面フロー
1. **ゲーム設定**
   - ウマ・オカの設定
   - 同点処理モードの選択

2. **点数入力**
   - 各プレイヤーの点数入力
   - リアルタイム計算結果表示

3. **結果表示**
   - 最終順位
   - 精算結果
   - 履歴保存

## 特徴的な機能
- 同点時の**上家取り**・**折半**処理
- 五捨六入による点数正規化
- 直感的な入力インターフェース
